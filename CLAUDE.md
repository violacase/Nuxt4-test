## E2E Type Chain

This is the core architecture principle. Never break this chain.

```
Postgres (port 5432)
  → Drizzle schema        [/server/db/schema/*.ts]
    → GraphQL schema      [/server/graphql/schema/*.ts]
      → GraphQL Codegen   [auto-generates /app/types/gql.ts]
        → Vue components  [fully typed queries + mutations]
```

- **Never** write manual TypeScript types for DB entities — derive them from Drizzle
- **Never** write manual TypeScript types for GQL responses — let codegen handle it
- `npm run dev` starts both the Nuxt dev server and codegen:watch automatically via `concurrently`

## Folder Structure

Nuxt 4 uses `app/` as the source directory (`srcDir`). All Vue app code lives there.
The `~` alias resolves to `app/`. The `~~` alias resolves to the project root.
The `server/` directory always stays at the project root.

```
/
```

## Naming Conventions

### Files
- Components: `PascalCase.vue` — e.g. `UserCard.vue`, `PostEditor.vue`
- Composables: `camelCase.ts` prefixed with `use` — e.g. `useAuth.ts`, `usePost.ts`
- Stores: `camelCase.ts` — e.g. `settings.ts`, `user.ts`
- Server routes: `kebab-case.ts` — e.g. `user-profile.get.ts`
- Drizzle schema: `camelCase.ts` per domain — e.g. `users.ts`, `posts.ts`
- GraphQL resolvers: `camelCase.ts` per domain — e.g. `users.ts`, `posts.ts`

### TypeScript
- Types/Interfaces: `PascalCase` — e.g. `UserProfile`, `PostInput`
- Drizzle inferred types: `type User = typeof users.$inferSelect`
- GQL types: always imported from `~/types/gql.ts` (never redefined)

### Database
- Table names: `snake_case` plural — e.g. `users`, `blog_posts`
- Column names: `snake_case` — e.g. `created_at`, `user_id`
- Drizzle table variables: `camelCase` — e.g. `users`, `blogPosts`

### GraphQL
- Types: `PascalCase` — e.g. `User`, `BlogPost`
- Queries: `camelCase` — e.g. `getUser`, `listPosts`
- Mutations: `camelCase` verb-first — e.g. `createPost`, `updateUser`, `deletePost`
- Input types: `PascalCaseInput` — e.g. `CreatePostInput`, `UpdateUserInput`

## Component Pattern (reka-ui + shadcn-vue + Tailwind)

Always use `<script setup lang="ts">` — never Options API.
Always use `cn()` from `~/lib/utils` for conditional class merging.

```vue
<script setup lang="ts">
// 1. Imports (external → internal → types)
import { cn } from '~/lib/utils'
import { SomeRekaComponent } from 'reka-ui'
import type { SomeType } from '~/types/gql.ts'

// 2. Props with TypeScript interface (always include class for override)
interface Props {
  label: string
  variant?: 'default' | 'outline'
  class?: string
}
const props = withDefaults(defineProps<Props>(), {
  variant: 'default'
})

// 3. Emits
const emit = defineEmits<{
  submit: [value: string]
}>()

// 4. Composables / stores
const { t } = useI18n()

// 5. Reactive state
const isOpen = ref(false)

// 6. Computed — use cn() for class merging
const classes = computed(() =>
  cn(
    'base-classes rounded-md',
    props.variant === 'default' && 'bg-primary text-primary-foreground',
    props.variant === 'outline' && 'border border-border bg-transparent text-foreground',
    props.class,
  )
)
</script>

<template>
  <!-- Single root element preferred -->
</template>
```

### Color Token Reference

| Intent              | Tailwind class              |
|---------------------|-----------------------------|
| Page background     | `bg-background`             |
| Card / panel        | `bg-card`                   |
| Primary text        | `text-foreground`           |
| Secondary / meta    | `text-muted-foreground`     |
| Border              | `border-border`             |
| Action (blue)       | `bg-primary` / `text-primary` |
| Text on action bg   | `text-primary-foreground`   |
| Hover background    | `bg-accent`                 |
| Error / danger      | `text-destructive`          |
| Focus ring          | `ring-ring/30`              |

**Never** use `text-[--color-primary]` (old naming) — use `text-foreground` for text, `text-primary` for the action color.
**Never** use raw Tailwind colors (`text-blue-500`, `bg-gray-100`).

## Settings Store Pattern (localStorage)

```ts
// stores/settings.ts
export const useSettingsStore = defineStore('settings', () => {
  const theme = useLocalStorage<'light' | 'dark'>('settings:theme', 'light')
  const locale = useLocalStorage<string>('settings:locale', 'en')
  const layout = useLocalStorage<'compact' | 'comfortable'>('settings:layout', 'comfortable')

  function reset() {
    theme.value = 'light'
    locale.value = 'en'
    layout.value = 'comfortable'
  }

  return { theme, locale, layout, reset }
})
```

Keys always prefixed with `settings:` to namespace localStorage.

## Drizzle Schema Pattern

```ts
// server/db/schema/users.ts
import { pgTable, uuid, text, timestamp } from 'drizzle-orm/pg-core'

export const users = pgTable('users', {
  id: uuid('id').defaultRandom().primaryKey(),
  email: text('email').notNull().unique(),
  name: text('name').notNull(),
  createdAt: timestamp('created_at').defaultNow().notNull(),
  updatedAt: timestamp('updated_at').defaultNow().notNull(),
})

// Always export inferred types
export type User = typeof users.$inferSelect
export type NewUser = typeof users.$inferInsert
```

## GraphQL Resolver Pattern

```ts
// server/graphql/resolvers/users.ts
import { db } from '../db'
import { users } from '../db/schema/users'
import { eq } from 'drizzle-orm'
import type { User } from '../db/schema/users'

export const userResolvers = {
  Query: {
    getUser: async (_: unknown, { id }: { id: string }): Promise<User> => {
      const result = await db.select().from(users).where(eq(users.id, id))
      if (!result[0]) throw new Error('User not found')
      return result[0]
    },
  },
  Mutation: {
    updateUser: async (_: unknown, { id, input }: { id: string, input: Partial<User> }): Promise<User> => {
      const result = await db.update(users).set(input).where(eq(users.id, id)).returning()
      return result[0]
    },
  },
}
```

## Quill Component Pattern

Always wrap in `<ClientOnly>` — Quill requires the DOM.

```vue
<ClientOnly>
  <QuillEditor
    v-model:content="content"
    content-type="html"
    :options="quillOptions"
    @update:content="emit('update:modelValue', content)"
  />
  <template #fallback>
    <div class="h-40 animate-pulse bg-muted rounded-md" />
  </template>
</ClientOnly>
```

Content is stored as HTML string in Postgres (`text` column type).
Always sanitize HTML server-side before storing (use `sanitize-html`).

## i18n

- Default locale: `en`
- Locale files: `i18n/locales/en.json`, `i18n/locales/nl.json` etc. (`@nuxtjs/i18n` v10 resolves `langDir` relative to `<rootDir>/i18n/`)
- Always use `const { t } = useI18n()` in components
- Never hardcode user-facing strings in components
- **`@` must be escaped in locale values** — vue-i18n treats `@` as a linked-message prefix. Use `{'@'}` anywhere a literal `@` appears (e.g. email placeholders, connection strings). Example: `"you{'@'}example.com"` not `"you@example.com"`.

## Rules for Claude Code

- **Never** use Options API
- **Never** write types that can be inferred from Drizzle or GQL codegen
- **Never** use `any` — use `unknown` and narrow it
- **Never** hardcode connection strings — always use `useRuntimeConfig()`
- **Never** use raw Tailwind color classes (`text-blue-500`) — use semantic utilities
- **Never** create a `tailwind.config.js` — Tailwind v4 is configured entirely in CSS
- **Always** use `cn()` from `~/lib/utils` for conditional class merging
- **Always** wrap Quill in `<ClientOnly>`
- **Always** sanitize HTML from Quill before storing
- **Always** protect server routes/resolvers with `requireUserSession`
- **Always** namespace localStorage keys with `settings:`
- **Always** run `npm run codegen` after changing a GraphQL schema file
- **Always** run `npm run db:generate` after changing a Drizzle schema file
- **Prefer** Drizzle `returning()` over a second select after insert/update

## Design System

**Always read `design-system.md` before generating any component or page.**

This project uses a professional, structured, refined, minimal aesthetic. The full
specification lives in `design-system.md` at the project root. Key rules:

- **Fonts:** `font-serif` (DM Serif Display) for headings, `font-sans` (DM Sans) for UI, `font-mono` for code
- **Colors:** Semantic Tailwind utilities only — `text-foreground`, `bg-background`, `bg-card`, `text-primary`, etc. Never raw colors like `text-blue-500`. `text-primary` = action blue, NOT body text (that's `text-foreground`).
- **Spacing:** 4px base unit — multiples only (gap-2, p-4, p-6 etc.) — no odd values
- **Borders:** `border border-border` — cards always have a border
- **Radius:** inputs `rounded-sm` (4px), buttons/cards `rounded-md` (8px) — never `rounded-full` on cards
- **Shadows:** used minimally — never as a substitute for a border
- **Motion:** context-dependent — use `/design-component` skill for guidance
- **Buttons:** 3 variants only — `solid`, `outline`, `ghost` — one `solid` per view max
- **shadcn-vue:** add components via `npx shadcn-vue@latest add <component>` — they land in `app/components/ui/`

When generating any UI, use `/design-component` for design-first scaffolding.
When generating structure only (no visual design needed), use `/new-component`.
